User Guide

Intro

This short user guide should serve as a guided walkthrough the main features of the Bidar applications so that anyone can use the tool as soon as possible to its full potential.

The order in which the sections are presented is the order in which you should should follow them.

Log in

In the login page, you can enter your credentials(username, password) combination in order to access the bidar tool

Also from this page you can access the registration page by cliking on the 'Register here' link

Register

If you don't have an account you can create one by using an username/password combination

The registration page can be accessed from the main login page

What awesome stuff you can do with BIDAR? :)

Profile

The profile section of the frontend application allows the user to update his first name, last name, age, country and his interests and skills For the interests and skills side of things, a list will be dynamically generated based on the data returned by the backend application for the current user. Also, the application allows the user to add new skills/interests and to delete current ones if he's not happy with them or if he's not interested in them anymore. Moreover, the application will display a warning text if the user hasn't enetered any skills or interests

Navigation

Since the frontend application is a SPA, the current page is updated based on the route take by the user through the navigation panel. The navigation is done, as stated above, through a side bar, which updated it's layout based on the current screen's dimensions in order to allow the user to use the application even on devices with smaller screens, e.g: smartphone

Main UI - Explorer

The explore section of the frontend application allows the user to search dbpedia for new resources, but also to see the resoruces the system returned for his current set of skills and interests. In addition, this section will have, at the bottom of the page, all the resources the system recommends for the user based on his metadata and based on the current resources too. As a small UX touch, this section displays the number of search results and the number of resources shown as recommendation to the user.

For each resource, a card like structure will be rendered, show:

  • A label
  • Resource's abstract
  • A link to the dbpedia for that specific resource

Also, the user has the possibility to limit the number of resources by choosing from a predefined list of limits that can be applied to the search
